# WAAS Wall UI Guide

This watchboard should stay readable under hackathon demo pressure.

Design direction used for this pass:

- Lead with the clearest story first.
  The first screen should answer: what is WAAL, what is happening right now, and where are the highest-risk decisions.
- Use stronger grouping and fewer competing panels.
  Related information should live in obvious containers with visible spacing rhythm rather than many small equal-weight boxes.
- Make stale data explicit.
  Coordination dashboards should surface stale or missing feeds directly instead of pretending everything is current.
- Use typography as hierarchy, not decoration.
  Large display headlines, restrained body copy, and monospace metadata keep the board readable at a distance.
- Reserve saturated color for meaning.
  Teal is healthy/active, amber is warning, red is blocked/risky. Neutral surfaces carry everything else.
